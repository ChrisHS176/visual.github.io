¿Qué es ADO.NET?

Definición

ADO.NET es la evolución del modelo de objetos ADO (ActiveX Data Objects) de Microsoft. Es el conjunto de clases, interfaces, estructuras y enumeraciones que permiten el acceso a datos desde la plataforma .NET.

💡 Clave: ADO.NET usa XML como formato universal de transmisión de datos y trabaja en modo desconectado para reducir tráfico de red.
Beneficios
BENEFICIO 1

🔄 Interoperabilidad

Usa XML como estándar, por lo que cualquier componente que interprete XML puede acceder a los datos de ADO.NET.

BENEFICIO 2

🔧 Mantenimiento

Permite cambios estructurales en el sistema sin la complejidad que tenían los modelos anteriores.

BENEFICIO 3

📝 Programación

API de fuerte tipificado, estructurado y centrado en la presentación correcta de datos.

BENEFICIO 4

⚡ Rendimiento

No se necesita conversión de tipos COM. Reduce ancho de banda e independiza cliente del servidor.

BENEFICIO 5

📈 Escalabilidad

Modo Off-Line evita reservar recursos innecesariamente. Más usuarios por unidad de tiempo. Soporta Clusters de alta disponibilidad.

Modo Desconectado

¿Cómo funciona la arquitectura desconectada?

ADO.NET se conecta al servidor solo lo estrictamente necesario para cargar los datos en el DataSet. Luego se desconecta y el programa trabaja con los datos locales.

Persistencia Off-Line

La persistencia permite que un componente guarde su estado (variables, propiedades, datos) en un momento dado. En ADO.NET puedes cerrar la app y mantener los DataSets guardados, de modo que al reiniciar los encuentras tal como los dejaste — sin necesidad de volver a consultar la BD.

Código Original
Form1.vb — VB.NET + OleDb 
Imports System.Data.OleDb

Public Class Form1
    Private Sub btnpeliculas_Click(sender As Object, e As EventArgs) Handles btnpeliculas.Click
        Dim conexion As String
        conexion = "Provider=Microsoft.ACE.OLEDB.12.0; Data Source=" & "IMDb.accdb"

        Dim seleccion = "SELECT * FROM Peliculas"

        Dim adaptadordedatos As New OleDbDataAdapter
        Dim tabladedatos As New DataTable

        Try
            adaptadordedatos = New OleDbDataAdapter(seleccion, conexion)
            adaptadordedatos.Fill(tabladedatos)
            MsgBox.DataSource = tabladedatos
        Catch ex As Exception
            Interaction.MsgBox("Error al mostrar los datos: " & ex.Message)
        End Try
    End Sub
End Class
Análisis Línea a Línea
1

Imports System.Data.OleDb

Importa el espacio de nombres que contiene las clases para conectarse a bases de datos a través de OleDB, como OleDbDataAdapter y OleDbConnection. Equivalente a un using en C#.

2

Cadena de conexión (conexion)

La cadena usa el proveedor Microsoft.ACE.OLEDB.12.0 (para archivos .accdb de Microsoft Access) y apunta al archivo IMDb.accdb. En producción, la ruta suele ser absoluta o configurada externamente.

3

Sentencia SQL (seleccion)

SELECT * FROM Peliculas recupera todas las columnas y filas de la tabla Peliculas. Ésta es la propiedad CommandText que el DataAdapter usará internamente.

4

OleDbDataAdapter

El DataAdapter es el puente entre la BD y el DataSet/DataTable. Abre la conexión, ejecuta el SELECT, llena la tabla y cierra la conexión automáticamente — todo en una sola operación con Fill().

5

DataTable (tabladedatos)

Almacén local de los datos. Después de Fill(), contiene todos los registros de la tabla Peliculas en memoria, sin necesidad de seguir conectado a la BD.

6

adaptadordedatos.Fill(tabladedatos)

Ejecuta la consulta SQL, transfiere los resultados al DataTable y cierra la conexión. Este es el paso central del modo desconectado.

7

MsgBox.DataSource = tabladedatos

MsgBox es el nombre asignado al control DataGridView en el formulario. Al asignarle el DataTable como DataSource, el grid muestra automáticamente todos los registros de la tabla Peliculas en pantalla.

8

Try / Catch

Manejo de errores. Si falla la conexión, la consulta u otra operación, el bloque Catch captura la excepción y muestra el mensaje de error con Interaction.MsgBox().

Clase Connection

Establecer la Conexión

Es el primer paso obligado. Existen dos implementaciones:

SqlConnection → SQL Server OleDbConnection → OleDB genérico
El método Open() abre la conexión definida en ConnectionString. Las conexiones también pueden abrirse implícitamente mediante un DataAdapter.

Clase DataReader

Lectura Rápida de Registros

Permite navegar hacia adelante y sólo lectura por los registros devueltos. Es el equivalente al Recordset de ADO en modo read-only / forward-only. Implementaciones: SqlDataReader y OleDbDataReader.

⚠️ El DataReader permanece conectado mientras recorre los registros. Es más rápido que el DataSet pero requiere conexión constante.
